# Directive 04: Navigational Engineering & Semantic Anchoring

**Zone 1 Summary:** This document instructs the agent to install the navigational layer on top of sanitized Pillar content — transforming clean files into a precision-retrieval knowledge graph. Its authority is Primary Source for all Zone 1 Identity Block drafting, Semantic Magnet header rewrites, Synthesis Bridge injection, and Linguistic Mirroring decisions. It is executed only after Gate 3 is cleared and governs production of the Navigational Engineering Report, presented for approval before any file is modified. This document answers: (1) What is the correct Zone 1 Identity Block for each of the 8 Pillar files? (2) Which generic headers must be rewritten as Semantic Magnets, and what should the new headers say? (3) Where must Synthesis Bridges be injected to enable cross-pillar reasoning?

## Step 1 — Internalize the Four Engineering Rules

Apply all four rules to every Pillar file:

### Rule 1 — The Zone 1 Identity Block
Every Pillar file must open with a structured YAML-style metadata block followed immediately by a bolded Zone 1 Summary of 100–150 words. This block is the first content the retrieval engine reads. It must answer four questions without ambiguity:
1. What is this document?
2. What specific questions will it answer?
3. What is its domain authority?
4. What will it NOT answer — and where should the agent look instead?

### Rule 2 — Semantic Magnetism
Every H2 and H3 header is a retrieval beacon. Replace any generic header with a keyword-rich, domain-specific descriptor that names exactly what the section contains.
- ❌ `## Theory` → ✅ `## Russell's Two-Way Light Principle — Thrust/Recoil Wave Dynamics`
- ❌ `## Calculations` → ✅ `## Torque Equations for 1.618 / √2 / √3 Geometric Balance Points`
- ❌ `## Introduction` → ✅ `## Quarter Theory Overview — Fractal Time Division from Quadral to 23-Minute Levels`

### Rule 3 — Cross-Document Synthesis Bridges
At every section break where one Pillar's content logically continues in another Pillar, inject a "See Also" pointer. Format:
`> **See Also:** For [next logical step], consult [CANONICAL_FILENAME.md] §[Section Name].`

### Rule 4 — Linguistic Mirroring
Every term used in a header or section title must match the canonical vocabulary in `REFERENCE_GLOSSARY.md`. If a header uses a synonym or variant, rewrite it to use the canonical term.

## Step 3 — Build the Semantic Magnet Transformation Table

Review every H2 and H3 header across all 8 Pillar files. For each generic or weak header, produce a rewrite.

A header is **weak** if it:
- Uses a single generic word (`## Theory`, `## Data`, `## Notes`, `## Introduction`)
- Does not name a specific concept, formula, or method
- Could belong to any document in any domain

A header **passes** if it:
- Contains domain-specific terminology
- Names exactly what the section covers without needing to read the section
- Would appear as a relevant result for a domain-specific query
